# 同类型项目对照整合计划

## 1. 可照搬与不可照搬结论

### 1.1 可直接照搬（数据结构/交互策略层）
- 多 API 配置集合 + 活跃配置切换
- 历史记录搜索与筛选交互模式
- 场景管理页的模板 CRUD 交互流
- Step4 流式脚本生成的渐进式渲染体验
- 全局任务状态条（任务类型 + loading + error）

### 1.2 不可直接照搬（代码层）
- React/Next 组件代码（当前项目是 Vue3 + Pinia）
- 浏览器直连大模型 SDK 代码（安全边界不符合当前桌面架构）
- 对方 Rust `main.rs`（能力过弱，很多 placeholder）

### 1.3 当前项目可重构复用逻辑
- `history + session snapshot` 合并为统一“会话中心”
- `rule prompt` 升级为 `system/user` 双模板，支持变量检查
- `taskStatus` 扩展为可取消任务（后端 run_id + cancel token）

## 3. 执行问题与风险

## 3.1 架构风险
- 对方项目是 Next + React，不能直接复制 TSX。
- 当前项目已有中文编码污染痕迹，新增改造需避免继续污染并逐步清理。

## 3.2 安全风险
- 不能照搬浏览器端 `dangerouslyAllowBrowser` 直连模型。
- API key 仍是本地存储，需要后续补充加密或系统密钥链方案。

## 3.3 运行风险
- 部分 provider（如 SiliconFlow）在浏览器模式下可能受 CORS 限制。
- 流式生成需要后端支持 chunk/event，当前命令是一次性返回。
