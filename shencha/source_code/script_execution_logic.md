# 脚本执行逻辑详解

## 整体流程

这个脚本系统主要是用来从各种格式的文件中提取结构化数据的。让我用通俗易懂的语言来解释整个执行过程：

### 1. 准备阶段

1. **创建测试数据**：脚本会自动创建一些测试文件，包括：
   - `symptom_1.txt`：包含发动机故障症状的文本文件
   - `symptom_2.md`：Markdown格式的症状表格
   - `symptom_3.csv`：CSV格式的症状数据
   - `symptom_4.json`：JSON格式的症状数据

2. **加载场景模板**：脚本会读取 `scene_templates/symptom_table.json` 文件，这个文件定义了数据提取的规则和结构。

3. **生成配置和提取脚本**：根据场景模板，脚本会生成：
   - 分析配置：包含字段定义、别名映射等
   - 适配配置：包含字段匹配模式等
   - 提取脚本：基于模板生成的Python脚本，用于实际执行数据提取

### 2. 执行阶段

1. **调用运行器**：主脚本调用 `runner.py` 来执行提取操作

2. **加载提取脚本**：运行器加载生成的提取脚本

3. **读取输入文件**：提取脚本会处理指定目录下的所有支持的文件类型：
   - PDF文件（使用pypdf或pdfplumber）
   - Word文档（使用python-docx）
   - Excel和CSV文件（使用pandas）
   - JSON文件
   - 文本文件

4. **解析内容**：提取脚本会：
   - 首先尝试识别表格格式的数据
   - 如果不是表格，尝试识别键值对格式
   - 如果都不是，尝试识别编号步骤格式
   - 如果还是不行，就把整个内容作为一个记录

5. **映射字段**：根据配置的字段和别名，将提取的数据映射到正确的字段

6. **生成输出**：将提取的数据以不同格式输出：
   - JSON格式
   - CSV格式
   - Markdown格式
   - Excel格式

## 可能遇到的问题及解决方案

### 1. 依赖库缺失

**问题**：执行脚本时出现 "ModuleNotFoundError" 错误

**原因**：缺少必要的Python库，如pypdf、pdfplumber、python-docx、pandas等

**解决方案**：
- 安装所需依赖：`pip install pypdf pdfplumber python-docx pandas openpyxl`
- 如果安装失败，可以使用 `python_runtime/requirements.txt` 文件安装：`pip install -r python_runtime/requirements.txt`

### 2. 文件格式不支持

**问题**：某些文件没有被处理

**原因**：脚本只支持特定格式的文件（PDF、DOCX、XLSX、XLS、CSV、MD、JSON、TXT）

**解决方案**：将文件转换为支持的格式，或者修改脚本添加对其他格式的支持

### 3. 数据提取失败

**问题**：提取的结果为空或不完整

**原因**：
- 文件格式不符合预期
- 字段映射规则不匹配
- 文件内容格式不规范

**解决方案**：
- 检查输入文件格式是否正确
- 检查场景模板配置是否合理
- 确保文件内容按照预期格式组织

### 4. 输出目录权限问题

**问题**：无法写入输出文件

**原因**：输出目录没有写入权限

**解决方案**：
- 确保输出目录存在且有写入权限
- 可以尝试修改输出目录为当前用户有权限的位置

### 5. 编码问题

**问题**：文件读取时出现编码错误

**原因**：文件使用了不支持的编码格式

**解决方案**：脚本已经处理了编码问题，会自动尝试使用UTF-8编码读取文件，如果失败会忽略错误

## 执行结果

执行完成后，脚本会在 `python_runtime/generated` 目录中生成：
- `flow_test_extract.py`：生成的提取脚本
- `flow_test_config.json`：配置文件
- 各种格式的输出文件：`extract_result_*.json`、`extract_result_*.csv`、`extract_result_*.md`、`extract_result_*.xlsx`

## 总结

这个脚本系统的核心功能是：
1. 从多种格式的文件中提取结构化数据
2. 支持自动识别不同的数据格式
3. 可以将提取的数据以多种格式输出
4. 具有一定的容错能力，即使遇到格式不规范的文件也能尝试提取

通过这个系统，你可以方便地从各种文档中提取结构化数据，无需手动复制粘贴，大大提高了工作效率。
