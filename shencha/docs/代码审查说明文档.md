# AI 脚本生成器 代码审查说明文档

## 项目概述

**项目名称**: AI 脚本生成器（Tauri + Vue3 + TS + Rust + Python）

**项目简介**: 基于"样本文本驱动"的 5 步工具，用于从各种格式的文件中提取结构化数据。

### 技术栈
- 桌面端：Tauri 2（Rust）
- 前端：Vue 3 + TypeScript + Pinia + Vue Router
- 脚本运行：Python 3
- 模板：本地 JSON 场景模板

### 核心能力
- 5 步严格流程导航（可回退、不可跳步）
- 场景模板系统（`scene_templates/*.json`）
- 第 3 步规则分析：普通分析 / 流式分析 / 提示词模板可编辑
- 第 4 步脚本生成：流式脚本展示 / 一键复制 / 下载 `extract.py` / `config.json`
- 第 5 步运行：选择输入路径/输出目录 / 执行 Python / 实时日志回传
- API 管理：多配置管理 / API 连通性测试
- 会话管理：记录各步骤进度和事件 / 可切回历史会话

## 前端交互逻辑说明

### 5步流程概览

#### 步骤1：输入源获取（Step1SourceView.vue）
**功能**: 获取用户要处理的原始文本

**交互逻辑**:
1. 用户点击"读取选中内容" → 调用 Tauri 命令 `read_selected_text`
2. 失败时点击"使用剪贴板内容" → 调用 `read_clipboard_text`
3. 样本不能为空才能进入下一步
4. 支持手动编辑样本文本
5. 可清空样本重置流程

**关键代码位置**: [Step1SourceView.vue](d:\aiprd-jiaoben\src\views\Step1SourceView.vue)

#### 步骤2：场景选择（Step2SceneView.vue）
**功能**: 选择数据抽取的业务场景

**交互逻辑**:
1. 选择一级场景（9种主场景）
2. 若选择"维修方案步骤抽取"，需再选子场景（5种子场景）
3. 支持"智能推荐场景"功能
4. 选择后自动创建会话记录
5. 样本预览显示当前样本文本

**场景类型**:
- `dtc_extraction`: DTC 故障码抽取
- `symptom_table`: 故障症状表抽取
- `check_confirm_text`: 检查与确认抽取
- `visual_inspection_table`: 外观检查表抽取
- `warning_notice`: 警告与注意提示抽取
- `diagnostic_flow`: 诊断流程表抽取
- `specification_table`: 规格参数表抽取
- `torque_spec`: 扭矩规格表抽取
- `repair_plan_steps`: 维修方案步骤抽取（含5子场景）

**关键代码位置**: [Step2SceneView.vue](d:\aiprd-jiaoben\src\views\Step2SceneView.vue)

#### 步骤3：规则分析（Step3RuleView.vue）
**功能**: AI 分析样本，生成结构化规则分析 JSON

**交互逻辑**:
1. 点击"生成规则分析"或"流式分析"
2. 可编辑 Prompt 模板（占位符：`{{selected_text}}`, `{{primary_scene}}`, `{{sub_scene}}`, `{{template_json}}`）
3. 可使用 AI 优化提示词
4. 输出 `analysis_json` 和 `analysis_basis`
5. 支持人工调整 `field_alias_map`

**关键代码位置**: [Step3RuleView.vue](d:\aiprd-jiaoben\src\views\Step3RuleView.vue)

#### 步骤4：脚本输出（Step4ScriptView.vue）
**功能**: 基于规则分析生成 Python 提取脚本

**交互逻辑**:
1. 输入：第3步分析结果 + 当前场景 schema
2. 输出：`extract.py` + `config.json`
3. 支持流式显示长脚本
4. 可一键复制脚本到剪贴板
5. 可下载脚本与配置文件

**关键代码位置**: [Step4ScriptView.vue](d:\aiprd-jiaoben\src\views\Step4ScriptView.vue)

#### 步骤5：运行结果（Step5RunView.vue）
**功能**: 执行生成的 Python 脚本

**交互逻辑**:
1. 选择输入文件或输入目录
2. 选择输出目录
3. 选择输出格式（xlsx / csv / json / md）
4. 点击"运行脚本"
5. 查看实时日志
6. 显示输出文件路径

**关键代码位置**: [Step5RunView.vue](d:\aiprd-jiaoben\src\views\Step5RunView.vue)

#### 核心流程

**规则分析流程**:
1. 接收 `AnalyzeRulesRequest`（包含样本、场景、模板、LLM配置）
2. 构建提示词（支持覆盖默认提示词）
3. 判断是否有诊断流程场景的特殊约束
4. 如有 API Key 调用 `analyze_with_online_model`
5. 否则使用 `build_mock_analysis` 本地模拟
6. 执行 `enforce_rule_package` 确保输出符合模板 schema

**脚本生成流程**:
1. 接收 `GenerateScriptRequest`（包含分析结果、场景schema、样本等）
2. 调用 `build_scene_adaptation` 构建基础适配
3. 如有 API Key 调用 `refine_adaptation_with_online_model` 优化
4. 使用 `render_script` 将适配 JSON 嵌入 Python 脚手架
5. 生成 `config.json` 包含完整配置

**脚本执行流程**:
1. 在 `python_runtime/generated/` 目录创建带时间戳的脚本和配置
2. 调用 `runner.py` 执行提取
3. 捕获 stdout/stderr 日志
4. 解析 `OUTPUT_FILE::` 前缀获取输出文件路径

### 场景模板配置

每个场景模板（JSON）包含：
- `scene_id`: 场景唯一标识
- `scene_name`: 场景中文名称
- `output_schema`: 输出字段定义（field, type, required, description）
- `header_alias`: 表头别名映射
- `mapping_rules`: 字段映射规则
- `validation_rules`: 验证规则
- `fallback_strategy`: 低置信度处理策略
- `examples`: 输入输出示例

**关键代码位置**: [scene_templates/symptom_table.json](d:\aiprd-jiaoben\scene_templates\symptom_table.json)

## 脚本说明

### Python 提取脚本（extract_scaffold.py）

**功能**: 基于场景适配配置，从多种格式文件中提取结构化数据

**支持的文件格式**:
- PDF（pypdf / pdfplumber）
- Word（python-docx）
- Excel/CSV（pandas）
- JSON
- 文本文件

**提取逻辑**:
1. 首先尝试识别表格格式的数据
2. 如果不是表格，尝试识别键值对格式
3. 如果都不是，尝试识别编号步骤格式
4. 如果还是不行，把整个内容作为一个记录

### Python 运行器（runner.py）

**功能**: 执行生成的 extract.py

**命令行参数**:
- `--script`: 生成的 extract.py 路径
- `--config`: config.json 路径
- `--input`: 输入文件或目录
- `--output-dir`: 输出目录
- `--format`: 输出格式（xlsx/csv/json/md）

### 流程测试脚本（run_flow_test.py）

**功能**: 本地流程测试

## 常见问题排查

1. **点击按钮无反应**
   - 先看页面底部消息提示
   - 若在浏览器模式，切到 `npm run tauri:dev`

2. **API 测试失败**
   - 检查 Base URL 是否含 `/v1`
   - 检查模型名是否与供应商一致
   - 检查 Key 是否可用

3. **`invoke` 相关报错**
   - 说明当前不是 Tauri Runtime
   - 使用桌面模式启动：`npm run tauri:dev`

4. **Tauri 编译失败（Windows）**
   - 确认安装 VS Build Tools C++ 组件
   - 执行 `npm run tauri:check` 观察具体报错

5. **Python 执行失败**
   - 检查 Python 版本与依赖
   - 检查输入路径和输出目录权限
